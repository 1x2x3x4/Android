# 定位校验设计说明

## 1. 设计目标

校园考勤系统需要体现“在校园范围内打卡”的业务特征。为了控制实现复杂度，本项目采用轻量级定位校验方案，不接入地图 SDK，也不做后台定位。

## 2. 权限策略

系统只在用户点击签到时检查位置权限，不默认请求后台定位权限。使用的权限包括：

```text
ACCESS_FINE_LOCATION
ACCESS_COARSE_LOCATION
```

如果用户未授权，系统会提示用户允许位置权限后再签到。

## 3. 校园范围规则

系统在设置页中提供以下配置项：

1. 校园中心纬度。
2. 校园中心经度。
3. 允许打卡半径。
4. 是否开启校园范围校验。

当校验开启时，系统获取设备最后一次可用位置，并计算当前位置与校园中心点之间的距离。如果距离大于允许半径，则禁止签到并提示“当前位置不在校园打卡范围内”。

## 4. 技术实现

定位逻辑封装在 `CampusLocationHelper` 中。距离计算使用 Android 原生 `Location.distanceBetween()` 方法完成，不依赖第三方地图服务。

## 5. 课程设计说明

该方案属于课程设计可展示版，重点体现校园场景和权限处理流程。后续如果扩展为真实系统，可接入地图 SDK、围栏服务或服务器端位置校验。
