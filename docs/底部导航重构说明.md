# 底部导航重构说明

## 1. 重构原因

原版本中首页、记录页、设置页和个人中心主要通过按钮跳转到不同 Activity。虽然功能完整，但页面之间较分散，演示时需要频繁进入和返回。为了更符合常见移动应用交互习惯，本次将主界面重构为类似 QQ、微信的底部导航结构。

## 2. 四个顶层页面

底部导航包含 4 个顶层页面：

| Tab | Fragment | 职责 |
| --- | --- | --- |
| 首页 | HomeFragment | 今日考勤状态、签到签退、统计概览、最近 7 天 |
| 记录 | RecordFragment | 历史记录、日期筛选、异常筛选、详情和备注 |
| 设置 | SettingsFragment | 考勤规则、提醒、定位校验、演示数据 |
| 我的 | ProfileFragment | 当前用户、修改密码、退出登录、项目信息 |

## 3. Activity 与 Fragment 分工

登录页和注册页仍保留为独立 Activity。登录成功后进入 MainActivity。MainActivity 只作为主壳页面，负责 BottomNavigationView 和 Fragment 容器，不再直接承载具体业务。

具体业务逻辑迁移到 Fragment 中：

```text
MainActivity
├── HomeFragment
├── RecordFragment
├── SettingsFragment
└── ProfileFragment
```

## 4. 体验提升

重构后，用户可以通过底部导航直接切换首页、记录、设置和我的页面，不需要频繁返回。MainActivity 使用 Fragment show/hide 方式缓存页面，能够尽量保留记录页筛选输入等状态，切换体验更自然。

## 5. 答辩展示价值

底部导航让项目结构更像真实 App。答辩时可以按“首页查看状态、记录查看数据、设置配置规则、我的退出登录”的顺序演示，业务主线更清晰，页面层级也更容易讲解。
