# 测试用例说明

## 1. 测试目标

测试目标是验证 CampusAttendance 是否满足课程设计的核心功能要求，包括注册、登录、签到、签退、历史记录、退出登录和提醒功能。

## 2. 功能测试用例

| 编号 | 测试内容 | 操作步骤 | 预期结果 |
| --- | --- | --- | --- |
| TC-01 | 空用户名注册 | 打开注册页，不输入用户名，点击注册 | Toast 提示请输入用户名 |
| TC-02 | 密码不一致注册 | 输入用户名，两次密码不同，点击注册 | Toast 提示两次输入的密码不一致 |
| TC-03 | 正常注册 | 输入新用户名和一致密码，点击注册 | 注册成功并返回登录页 |
| TC-04 | 重复注册 | 使用已注册用户名再次注册 | Toast 提示用户名已存在 |
| TC-05 | 错误密码登录 | 输入正确用户名和错误密码登录 | Toast 提示用户名或密码错误 |
| TC-06 | 正常登录 | 输入正确用户名和密码登录 | 进入首页，显示欢迎信息 |
| TC-07 | 登录状态保持 | 登录成功后关闭应用再打开 | 启动页后自动进入首页 |
| TC-08 | 正常签到 | 登录后点击签到 | 首页显示签到时间，状态为已签到 |
| TC-09 | 重复签到 | 同一天再次点击签到 | Toast 提示今天已经签到过了 |
| TC-10 | 未签到签退 | 新用户未签到时点击签退 | Toast 提示请先完成签到再签退 |
| TC-11 | 正常签退 | 签到后点击签退 | 首页显示签退时间，状态为已完成 |
| TC-12 | 重复签退 | 同一天再次点击签退 | Toast 提示今天已经签退过了 |
| TC-13 | 查看历史记录 | 完成签到后进入历史记录页 | RecyclerView 显示日期、签到、签退、状态 |
| TC-14 | 空记录显示 | 新用户未打卡时进入历史记录页 | 显示暂无打卡记录 |
| TC-15 | 退出登录 | 进入个人中心点击退出登录 | 清除登录状态并返回登录页 |
| TC-16 | 通知提醒 | 首页点击演示打卡提醒 | 立即显示通知，并在约 10 秒后再次提醒 |
| TC-17 | 修改密码成功 | 个人中心进入修改密码页，输入正确原密码和新密码 | 提示密码修改成功 |
| TC-18 | 原密码错误 | 修改密码时输入错误原密码 | 提示原密码错误 |
| TC-19 | 月度统计刷新 | 完成签到或签退后返回首页 | 本月统计数据刷新 |
| TC-20 | 每日提醒开关 | 点击开启或关闭每日 8:00 提醒 | 按钮状态和提醒设置同步变化 |

## 3. 数据持久化测试

1. 注册用户后关闭应用，再次打开应用，用户仍可登录。
2. 完成签到或签退后关闭应用，再次登录，历史记录仍存在。
3. 退出登录后再次打开应用，应进入登录页，而不是直接进入首页。

## 4. 异常输入测试

系统对以下情况进行了 Toast 提示：

1. 用户名为空。
2. 密码为空。
3. 两次密码不一致。
4. 用户名重复。
5. 登录账号或密码错误。
6. 重复签到。
7. 未签到就签退。
8. 重复签退。

## 5. 构建验证

在项目根目录执行以下命令：

```bash
./gradlew :app:assembleDebug
```

如果输出 `BUILD SUCCESSFUL`，说明项目可以正常编译，并会生成 Debug APK。
