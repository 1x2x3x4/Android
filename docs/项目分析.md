# 项目分析

## 1. 开发背景

校园日常教学管理中，学生到课和离校记录是常见管理需求。传统纸质点名效率较低，记录不便统计，异常情况也不易追踪。本项目设计并实现一个基于 Android 的校园考勤打卡应用，用移动端完成注册登录、签到、签退、记录查询、统计分析和提醒等功能。

## 2. 应用定位

CampusAttendance 定位为轻量级本地校园考勤应用，主要用于课程设计演示和基础 Android 技术综合训练。项目不依赖服务器，数据存储在本机 SQLite 和 SharedPreferences 中，便于安装、运行和答辩展示。

## 3. 目标用户

目标用户为学生用户。学生可以注册账号、登录系统、按照考勤规则完成签到和签退，并查看自己的历史记录、异常记录和统计情况。

## 4. 功能需求

系统主要功能包括：

1. 用户注册、登录、记住登录状态、退出登录。
2. 每日签到、每日签退、重复打卡限制。
3. 按时间规则判定正常、迟到、早退、迟到早退和缺卡。
4. 首页展示今日状态、本月统计和最近 7 天考勤情况。
5. 历史记录查询、日期筛选、异常筛选和详情查看。
6. 异常记录备注和申诉预留。
7. 考勤规则、提醒和校园定位范围配置。
8. 基于通知的签到和签退提醒。
9. 答辩演示数据生成和清空。

## 5. 非功能需求

系统要求界面简洁、操作流程清晰、数据本地持久化、构建运行稳定。技术上保持 Java + XML + SQLite + SharedPreferences 的基础实现方式，避免引入超出课程要求的复杂框架。

## 6. 核心业务流程

核心流程为：首次打开应用进入登录页，未注册用户先注册；登录后进入 MainActivity 主界面。主界面使用底部导航栏划分为“首页、记录、设置、我的”四个顶层页面。用户在首页查看今日规则和状态并完成签到签退，在记录页筛选和查看详情，在设置页配置考勤时间、提醒和校园范围校验，在我的页管理账号。

## 7. 技术栈与开发环境

| 项目 | 内容 |
| --- | --- |
| IDE | Android Studio |
| 语言 | Java |
| UI | Android XML 原生布局 |
| 主界面导航 | BottomNavigationView + Fragment |
| 构建 | Gradle |
| 本地数据库 | SQLiteOpenHelper |
| 轻量配置 | SharedPreferences |
| 列表控件 | RecyclerView |
| 提醒 | AlarmManager、BroadcastReceiver、Notification |
| 定位 | Android LocationManager |
| 最低版本 | Android 7.0 API 24 |
