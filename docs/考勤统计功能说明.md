# 考勤统计功能说明

## 1. 功能目标

考勤系统不仅需要记录数据，还应提供基本分析能力。统计功能用于帮助学生了解自己的考勤表现，也能体现系统的数据处理价值。

## 2. 统计内容

首页当前展示以下统计信息：

1. 累计签到次数。
2. 累计正常次数。
3. 累计迟到次数。
4. 累计早退次数。
5. 累计缺卡次数。
6. 本月签到次数。
7. 本月正常次数。
8. 本月异常次数。
9. 最近 7 天考勤情况。

## 3. 技术实现

统计数据来源于 SQLite 中的 `attendance` 表。系统通过 `DatabaseHelper.getAttendanceStatistics()` 查询当前用户的全部记录，并根据 `status` 字段进行分类统计。

最近 7 天统计通过生成最近 7 个日期字符串，然后与数据库中的考勤日期匹配。如果某一天没有记录，则显示“无记录”。

## 4. 展示方式

为了保持课程设计实现简洁，当前版本使用 TextView 展示统计结果，没有引入复杂图表库。这样可以避免额外依赖，同时仍能清晰展示统计逻辑。

## 5. 应用价值

统计功能使系统从“单纯打卡工具”升级为“考勤分析工具”，体现了本地数据库查询、条件筛选和数据汇总能力。
