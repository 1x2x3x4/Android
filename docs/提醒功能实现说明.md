# 提醒功能实现说明

## 1. 功能目标

提醒功能用于帮助学生按时完成签到和签退。基础版只支持点击按钮立即通知，强化版增加了固定时间提醒和重启恢复能力。

## 2. 当前功能

系统当前支持：

1. 点击首页按钮立即发送演示通知。
2. 安排 10 秒后的 BroadcastReceiver 演示提醒。
3. 开启每日签到提醒。
4. 开启每日签退提醒。
5. 设备重启后恢复已开启的提醒任务。
6. Android 13 及以上系统运行时申请通知权限。

## 3. 技术实现

提醒功能由 `ReminderHelper` 统一管理。系统使用 AlarmManager 注册重复提醒任务，使用 BroadcastReceiver 接收提醒广播，再通过 Notification 发送系统通知。

涉及类如下：

| 类名 | 作用 |
| --- | --- |
| ReminderHelper | 创建通知、注册和取消提醒 |
| AttendanceReminderReceiver | 接收提醒广播并发送通知 |
| BootReceiver | 设备重启后恢复提醒 |
| AttendanceSettingsActivity | 设置提醒开关 |

## 4. 数据保存

提醒开关状态和提醒时间保存到 SharedPreferences 中。这样应用关闭或设备重启后，系统仍能读取用户设置并恢复提醒任务。

## 5. 实现效果

强化后的提醒功能更接近真实考勤系统：学生可以设置签到和签退提醒，系统会在固定时间主动提示，避免漏签和忘记签退。
