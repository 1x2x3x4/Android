# 后续完善功能说明

## 1. 修改密码功能

本次完善将个人中心中的“修改密码”入口由预留功能改为可用功能。

新增页面：

```text
ChangePasswordActivity
activity_change_password.xml
```

用户需要输入原密码、新密码和确认新密码。系统会完成以下校验：

1. 原密码不能为空。
2. 新密码不能为空。
3. 新密码至少 4 位。
4. 两次新密码必须一致。
5. 新密码不能与原密码相同。
6. 原密码必须与数据库中的当前密码匹配。

校验通过后，调用 `DatabaseHelper.updatePassword()` 更新 `user` 表中的 password 字段。该功能补充了 SQLite 的更新操作示例，使数据库功能不只包含插入和查询，也包含修改。

## 2. 首页月度统计

首页新增“本月统计”信息，显示当前用户本月已签到天数和完整打卡天数。

核心查询方法为：

```text
DatabaseHelper.getMonthlySummary(username, monthPrefix)
```

其中 `monthPrefix` 格式为 `yyyy-MM`。数据库查询通过 `date LIKE 'yyyy-MM%'` 获取当月记录，再统计：

1. `check_in_time` 不为空的记录数量，作为已签到天数。
2. `status` 为“已完成”的记录数量，作为完整打卡天数。

该功能便于学生用户快速了解本月考勤情况，也可以作为课程报告中的扩展功能展示。

## 3. 每日固定提醒

原有提醒功能为点击按钮立即发送通知，并安排 10 秒后的演示提醒。本次完善新增每日 8:00 提醒开关。

涉及组件：

```text
ReminderHelper
AttendanceReminderReceiver
BootReceiver
```

实现逻辑如下：

1. 用户点击首页“开启每日 8:00 提醒”。
2. 系统将提醒状态保存到 SharedPreferences。
3. 使用 AlarmManager 注册每日 8:00 的重复提醒。
4. 到达提醒时间后，BroadcastReceiver 接收广播并发送通知。
5. 如果设备重启，BootReceiver 会读取提醒开关状态，并重新注册每日提醒。

该功能进一步覆盖了 SharedPreferences、AlarmManager、BroadcastReceiver 和 Notification 的组合使用。
