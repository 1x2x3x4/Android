# 详细设计与实现

## 1. 页面设计说明

系统主要页面包括启动页、登录页、注册页、MainActivity 主界面、修改密码页。MainActivity 中使用底部导航切换首页、记录、设置和我的四个 Fragment。页面采用 XML 原生布局，整体以简单清晰为主，方便课程演示。

## 1.1 底部导航实现

主界面使用 BottomNavigationView 和 Fragment 容器实现。MainActivity 缓存 HomeFragment、RecordFragment、SettingsFragment 和 ProfileFragment，通过 show/hide 切换页面，避免重复创建页面并尽量保留筛选状态。

## 2. 登录注册实现

注册时，RegisterActivity 校验用户名、密码和确认密码，调用 DatabaseHelper 写入 user 表。登录时，LoginActivity 调用 DatabaseHelper 校验账号密码，成功后通过 SessionManager 保存登录状态。

## 3. 考勤规则实现

考勤时间规则由 AttendanceRuleManager 管理，使用 SharedPreferences 保存。签到时系统判断是否早于签到开始时间、是否晚于签到截止时间；签退时系统判断是否早退、是否缺卡，并更新 attendance 表中的 status 和 remark 字段。

## 4. 历史记录实现

RecordFragment 使用 RecyclerView 展示考勤记录。用户可以按日期筛选，也可以勾选“仅查看异常记录”。点击记录后弹出详情窗口，显示日期、签到时间、签退时间、状态和备注，并支持保存备注。

## 5. 统计功能实现

统计逻辑由 DatabaseHelper.getAttendanceStatistics() 完成。系统遍历当前用户考勤记录，统计累计签到、正常、迟到、早退、缺卡、本月正常、本月异常和最近 7 天情况。首页使用 TextView 展示统计结果。

## 6. 提醒功能实现

ReminderHelper 使用 AlarmManager 注册签到和签退提醒，AttendanceReminderReceiver 接收广播并发送通知。BootReceiver 监听开机广播，设备重启后恢复已开启的提醒。Android 13 及以上系统会在运行时申请通知权限。

## 7. 定位校验实现

CampusLocationHelper 使用 Android 原生 LocationManager 获取当前位置，并通过 Location.distanceBetween() 计算当前位置与校园中心点的距离。当校园范围校验开启且用户不在范围内时，系统禁止签到。

## 8. Android 核心知识点

项目覆盖 Activity、Fragment、Intent、BottomNavigationView、RecyclerView、SQLiteOpenHelper、SharedPreferences、BroadcastReceiver、AlarmManager、Notification、运行时权限、Toast 事件反馈和 XML 布局等课程知识点。
