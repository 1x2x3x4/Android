# 系统测试

## 1. 测试环境

测试环境为 Android Studio + Gradle，最低适配 Android 7.0。项目通过命令 `./gradlew :app:assembleDebug` 构建 Debug APK。

## 3. 异常测试用例

系统已处理用户名为空、密码为空、注册密码不一致、登录失败、重复打卡、记录为空、筛选无结果、备注为空、位置权限拒绝、通知权限拒绝、非法时间设置、定位数据不可用等情况，并通过 Toast 或空状态文案反馈。

## 4. 测试结果

当前版本已通过 Gradle Debug 构建，底部导航四个页面可正常切换，主要业务流程可演示。数据库数据在应用重启后仍然保留，退出登录后再次登录可继续查看原有记录。

## 5. 问题与修复

开发过程中曾出现按钮文字受主题背景影响显示不清的问题，已统一改用 AppCompatButton 并关闭 backgroundTint。后续又补充了状态统一工具类，避免“已完成”和“正常”等旧状态显示不一致。

## 6. 性能与稳定性简述

项目数据量较小，SQLite 查询压力低。提醒使用 AlarmManager，定位只在签到时触发，不进行后台定位，因此对性能和电量影响较小。
