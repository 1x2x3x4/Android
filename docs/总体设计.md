# 总体设计

## 1. 系统架构说明

系统采用 Activity + Fragment + 工具类 + SQLiteOpenHelper 的朴素结构。LoginActivity 和 RegisterActivity 负责登录注册，MainActivity 作为底部导航容器，HomeFragment、RecordFragment、SettingsFragment、ProfileFragment 分别承载四个顶层页面。DatabaseHelper 负责数据库读写，SessionManager、AttendanceRuleManager、ReminderHelper 和 CampusLocationHelper 分别负责会话、规则、提醒和定位配置。

## 2. 功能模块划分

| 模块 | 主要类 | 说明 |
| --- | --- | --- |
| 用户模块 | LoginActivity、RegisterActivity、ProfileFragment | 注册、登录、个人中心和退出 |
| 考勤模块 | HomeFragment、DatabaseHelper | 签到、签退、状态判定 |
| 规则设置模块 | SettingsFragment、AttendanceRuleManager | 配置考勤时间规则 |
| 记录模块 | RecordFragment、AttendanceAdapter | 历史记录、筛选、详情和备注 |
| 统计模块 | AttendanceStatistics、DatabaseHelper | 累计统计、本月统计、最近 7 天 |
| 提醒模块 | ReminderHelper、AttendanceReminderReceiver、BootReceiver | 签到签退提醒和重启恢复 |
| 定位模块 | CampusLocationHelper | 校园范围校验 |

## 3. 页面跳转关系

应用启动后进入 SplashActivity。SplashActivity 根据 SharedPreferences 判断登录状态，已登录则进入 MainActivity，未登录则进入 LoginActivity。LoginActivity 可跳转到 RegisterActivity。MainActivity 内部使用 BottomNavigationView 切换 HomeFragment、RecordFragment、SettingsFragment 和 ProfileFragment。ProfileFragment 可跳转到 ChangePasswordActivity。

## 4. 数据存储方案

用户账号和考勤记录存储在 SQLite 数据库中。登录状态、考勤规则、提醒开关和校园定位配置存储在 SharedPreferences 中。这样既能保证核心业务数据持久化，又能使轻量配置读取方便。

## 5. 数据库表结构设计

`user` 表保存用户信息：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | INTEGER | 主键，自增 |
| username | TEXT | 用户名，唯一 |
| password | TEXT | 密码 |

`attendance` 表保存考勤记录：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | INTEGER | 主键，自增 |
| username | TEXT | 用户名 |
| date | TEXT | 日期 |
| check_in_time | TEXT | 签到时间 |
| check_out_time | TEXT | 签退时间 |
| status | TEXT | 考勤状态 |
| remark | TEXT | 备注 |

`attendance` 表对 `username + date` 设置唯一约束，保证同一用户同一天只有一条考勤记录。

## 6. 关键类职责说明

DatabaseHelper 负责用户注册、登录校验、考勤写入、状态更新、记录查询和统计。MainActivity 负责底部导航和 Fragment 切换。HomeFragment 负责首页展示、签到签退和统计刷新。RecordFragment 负责记录筛选、详情弹窗和备注保存。SettingsFragment 负责规则配置、提醒开关、定位配置和演示数据管理。
