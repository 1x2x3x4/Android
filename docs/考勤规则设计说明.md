# 考勤规则设计说明

## 1. 设计目标

基础版考勤系统只能记录按钮点击时间，业务真实性较弱。强化版在签到和签退流程中加入时间规则、次数限制、先后顺序限制和状态判定，使系统更接近真实校园考勤场景。

## 2. 核心规则

系统当前采用以下规则：

1. 同一用户同一天只能签到一次。
2. 同一用户同一天只能签退一次。
3. 未签到时不能直接签退。
4. 签到开始时间之前不允许签到。
5. 签到截止时间之后签到，判定为迟到。
6. 签退开始时间之前签退，判定为早退。
7. 签退截止时间之后仍未签退，判定为缺卡。

## 3. 技术实现

考勤规则由 `AttendanceRuleManager` 管理，规则时间保存到 SharedPreferences 中。默认规则如下：

| 规则项 | 默认值 |
| --- | --- |
| 签到开始时间 | 07:30 |
| 签到截止时间 | 08:10 |
| 签退允许开始时间 | 17:00 |
| 签退截止时间 | 22:00 |

签到和签退操作在 `MainActivity` 中触发，系统会读取当前时间并与规则时间比较，然后将结果写入 SQLite 的 `attendance` 表。

## 4. 实现效果

通过规则控制，系统不再只是简单记录点击动作，而是能够根据考勤制度自动判断迟到、早退、缺卡和正常等业务结果，提高了项目的真实性和可演示性。
