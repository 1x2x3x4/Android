# 考勤状态判定说明

## 1. 状态类型

系统主要使用以下考勤状态：

| 状态 | 含义 |
| --- | --- |
| 未签到 | 当天尚未完成签到 |
| 已签到 | 已完成签到，但尚未签退 |
| 正常 | 签到和签退均符合时间规则 |
| 迟到 | 签到时间晚于签到截止时间 |
| 早退 | 签退时间早于签退允许开始时间 |
| 迟到早退 | 同一天既迟到又早退 |
| 缺卡 | 超过签退截止时间仍未完成有效签退 |

## 2. 判定流程

签到时：

```text
当前时间 < 签到开始时间：禁止签到
当前时间 > 签到截止时间：记录为迟到
其他情况：记录为已签到
```

签退时：

```text
未签到：禁止签退
已签退：禁止重复签退
签退时间 < 签退允许开始时间：早退
签退时间 > 签退截止时间：缺卡
签到状态包含迟到且签退早于允许开始时间：迟到早退
其他情况：正常
```

## 3. 缺卡处理

系统在首页刷新时会检查当前用户是否存在超过签退截止时间仍未签退的记录。如果存在，系统会自动将该记录状态更新为“缺卡”，并在备注中写入“超过签退截止时间未签退”。

## 4. 数据保存

考勤状态保存在 SQLite 的 `attendance.status` 字段中，异常原因或补充说明保存在 `attendance.remark` 字段中。这样既能支持列表展示，也能支持统计分析和详情查看。
